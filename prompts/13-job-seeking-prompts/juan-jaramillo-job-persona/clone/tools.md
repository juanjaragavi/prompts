# Tools

## Purpose

This file defines the tools, systems, and working environments that Juan Jaramillo commonly uses or prefers, so that agents and AI collaborators can make better decisions about how to support him.

## Tool Philosophy

Juan uses tools to create leverage, not novelty.

A good tool should:

- Reduce friction
- Increase speed
- Improve quality
- Support reuse
- Strengthen clarity and execution
- Fit into a larger workflow

Tools are valuable when they help move from idea to deliverable faster and with better standards.

## Preferred Tool Categories

Juan works across several categories of tools:

- LLMs and AI assistants
- IDEs and AI coding environments
- Full-stack development frameworks
- Cloud and infrastructure platforms
- Documentation and writing systems
- Workflow automation systems
- Communication and professional outreach tools
- Research and analysis tools

## LLM and AI Assistant Preferences

Juan is highly comfortable working with advanced LLM systems for:

- Planning
- Drafting
- Prompt engineering
- Code generation
- Refactoring
- Research synthesis
- Workflow design
- Documentation

He prefers using AI as a structured collaborator rather than a casual chatbot.

### Model Usage Pattern

A strong working pattern is:

- Use a higher-reasoning model for planning, architecture, and complex thinking
- Use a faster coding-oriented model for implementation and iterative execution

This planning/coding split fits Juan’s preference for combining strategic reasoning with fast production output.

## Prompting Preferences

Juan prefers prompts and agent instructions that are:

- Explicit
- Structured
- Context-rich
- Modular
- Reusable
- Easy to version and refine

Preferred prompt design principles:

- State the role clearly
- Define constraints
- Specify output format
- Include tone requirements
- Give relevant context in files, not only in chat
- Prefer durable instructions over repeatedly re-explaining the same thing

However, output quality matters more than tool novelty. The end result must feel deliberate and high-signal.

## Tool Selection Rules

When choosing tools on Juan’s behalf:

- Prefer the simplest tool that can do the job well
- Prefer reusable systems over one-off hacks
- Prefer structured files over scattered chat history
- Prefer tools that support speed without lowering quality
- Prefer environments that reduce ambiguity and preserve context

## Tool Misuse to Avoid

Avoid using tools in ways that create:

- Fragmented context
- Redundant work
- Weak drafts
- Overcomplicated workflows
- Excessive dependence on ephemeral chat state
- Fancy outputs with low practical value

If a tool adds complexity without meaningful leverage, it should not be the default choice.

## Ideal Tool Behavior for Agents

An agent operating on Juan’s behalf should:

- Read available context first
- Use the right tool for the specific task
- Preserve important information in files
- Produce clean outputs
- Reduce repetitive manual work
- Maintain high professional standards

## Final Standard

A tool is a good fit for Juan when it helps him:

- Think better
- Build faster
- Communicate more clearly
- Maintain credibility
- Reuse knowledge across tasks
- Operate at a senior level with less friction
