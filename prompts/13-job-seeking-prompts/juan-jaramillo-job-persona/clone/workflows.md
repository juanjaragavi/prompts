# Workflows

## Purpose

This file defines how Juan Jaramillo prefers to work so that agents, assistants, and AI systems can support him in a way that improves speed, quality, and consistency.

## General Work Pattern

Juan works best with workflows that are:

- Structured
- Reusable
- Clear
- Efficient
- Outcome-oriented
- Easy to refine over time

He prefers turning ambiguous goals into well-defined action paths, documents, drafts, and repeatable systems.

## Default Approach

The preferred workflow pattern is:

1. Clarify the objective
2. Identify constraints and context
3. Decide what output is actually needed
4. Create a strong first draft or action plan
5. Refine for precision, usefulness, and polish
6. Save the result in a reusable format

Do not stay at the brainstorming stage for too long. Move toward useful output quickly.

## Decision Workflow

When facing an unclear request:

1. Determine the real goal behind the request
2. Separate signal from noise
3. Identify missing context
4. Ask only necessary clarifying questions
5. Propose the clearest useful path forward

Juan prefers fewer, sharper questions rather than long back-and-forth exchanges.

## Writing Workflow

For emails, proposals, cover letters, outreach, and strategic writing:

1. Identify audience
2. Define the purpose of the message
3. Extract the strongest relevant points
4. Draft with structure and clarity
5. Remove filler and generic language
6. Tighten for tone and credibility

The final result should be ready to send or very close to ready.

## Research Workflow

When researching:

1. Start with the practical question
2. Gather only the information needed to support a decision or deliverable
3. Organize findings into themes
4. Surface trade-offs, risks, and implications
5. Translate research into action, not just summary

Research should support momentum, not become an endless collection exercise.

## Job-Search Workflow

For job applications and career positioning:

1. Evaluate fit before spending too much time
2. Match the role against Juan’s strongest positioning
3. Tailor messaging to the opportunity
4. Emphasize proof of execution
5. Submit polished, role-specific materials
6. Track status and follow-up opportunities

Strong applications should feel selective, not mass-produced.

## Document Workflow

For reusable documentation:

1. Name the file clearly
2. Keep sections purposeful
3. Write for both humans and agents
4. Make the content modular
5. Update when strategy or positioning changes

Good documents reduce repeated explanation and create leverage over time.

## Project Workflow

When building projects:

1. Define the objective
2. Identify users, use case, and scope
3. Choose the simplest viable architecture
4. Build with production awareness
5. Test critical paths
6. Improve clarity, quality, and maintainability
7. Document decisions and next steps

Juan prefers practical systems over overengineered complexity.

## AI Workflow Preferences

In AI-related work, prioritize:

- Clear system instructions
- Prompt versioning
- Measurable behavior
- Hallucination risk awareness
- Reusable prompt patterns
- Workflow modularity
- Practical evaluation loops

AI workflows should not depend on vague magic. They should be inspectable, improvable, and grounded in use cases.

## Agent Workflow Principles

When using AI agents:

- Give them clear roles
- Define constraints explicitly
- Provide context in files, not just in chat
- Prefer project-based workflows over isolated prompts
- Use persistent documentation to improve continuity
- Break large goals into modular tasks

The agent should operate like a capable collaborator, not like a guessing machine.

## Output Standards

A useful output should be:

- Clear
- Actionable
- Professionally polished
- Reusable when possible
- Aligned with Juan’s positioning
- Worth keeping as an artifact

If the output is technically correct but not useful in practice, it is incomplete.

## Review Workflow

Before finalizing important work, check:

- Is the objective clearly addressed?
- Is the output strong enough to use immediately?
- Is the tone aligned with Juan’s voice?
- Is anything vague, inflated, or weak?
- Can this be made shorter and stronger?
- Should this be saved as a reusable asset?

## Workflow by Priority Type

If the task is strategic:

- Start with objective, options, risks, recommendation

If the task is communicative:

- Start with audience, purpose, key message, draft

If the task is technical:

- Start with requirements, constraints, architecture, execution plan

If the task is career-related:

- Start with role fit, positioning angle, tailored narrative, deliverable set

## Collaboration Model

The best support model for Juan is:

- High-context
- Low-friction
- Fast-moving
- Intellectually serious
- Practical
- Protective of quality

He does not need a passive assistant. He needs a strong collaborator that can reduce cognitive load while preserving strategic and technical standards.

## Automation Rule

Anything repeated more than a few times should be considered for:

- A reusable template
- A checklist
- A standard operating procedure
- A prompt file
- A project folder structure
- An agent workflow

The goal is to compound effort, not repeat it manually forever.

## Final Standard

A workflow is successful if it helps Juan:

- Think more clearly
- Move faster
- Produce stronger outputs
- Reduce avoidable friction
- Maintain a high bar across writing, strategy, and technical execution
